# <a name="microsoft-365-certification-faq"></a>Microsoft 365 Häufig gestellte Fragen zur Zertifizierung

## <a name="introduction"></a>Einführung
Nachfolgend finden Sie häufig gestellte Fragen (FAQs), die von ISVs (Independent Software Vendors) beim Starten der Microsoft 365-Zertifizierung gestellt werden.  Wenn Sie Abfragen haben, die hier nicht behandelt werden, wenden Sie sich über AppCert@Microsoft.com an das Microsoft 365 App-Zertifizierungsteam.
Dieses Dokument richtet sich an ISVs. Allgemeine Informationen zum Microsoft 365 Security and Compliance-Programm finden Sie auf der [Seite Microsoft 365 App Compliance-Programm.](https://docs.microsoft.com/microsoft-365-app-certification/docs/enterprise-app-certification-guide) 


|Frage                                       |Erweiterte Definition                          |
|----------------------------------------       |---------------------------------------------|
|Ich habe keine Prüfung mit einem branchenweit anerkannten Framework wie PCI DSS, SOC 2 oder ISO 27001 bestanden.  Bedeutet dies, dass ich mich nicht für die Microsoft 365-Zertifizierung bewerben kann? | Nein, das Abrufen eines dieser branchenweit anerkannten Frameworks ist keine Voraussetzung für die Microsoft 365 Zertifizierung.|
|Ich habe bereits eine externe Prüfung mit einem branchenweit anerkannten Framework bestanden. Kann dies auf die Microsoft 365-Zertifizierung angerechnet werden? | Die kurze Antwort lautet "Ja".  Derzeit akzeptiert die Spezifikation Microsoft 365 Zertifizierung Nachweise für externe PCI DSS-, SOC 2- und ISO27001-Frameworks. Das Handbuch für Microsoft 365 Zertifizierungsübermittlungen hat die Ausrichtung dieser vorhandenen externen Frameworks zugeordnet. Wir haben jedoch in einigen Fällen festgestellt, dass der vorhandene Standard/Framework nicht angemessen ausgerichtet wurde. Aus diesem Grund führt das Microsoft 365-Zertifizierungsteam eine Überprüfung der bereitgestellten Standards/Framework-Nachweise durch und markiert, welche Kontrollen innerhalb der Microsoft 365 Zertifizierung erfüllt sind. |
|Wie demonstrieren wir die DSGVO-Compliance, wenn wir keine externe DSGVO-Bewertung durchgeführt haben? | Microsoft erfordert keine unabhängige Überprüfung der DSGVO-Konformität für die Microsoft 365-Zertifizierung, da dies entweder/oder ein Szenario ist, in dem wir Selbstnachweise akzeptieren, die wir unabhängig überprüfen können, wenn keine externe Überprüfung stattgefunden hat. Da es sich hierbei eher um eine Bewertung als um eine Prüfung handelt, und im Hinblick auf die Nachweise, die wir während unserer Prozesse sammeln müssen, ist die Überprüfung von Datenschutzrichtlinien und internen Prozessen die Vorgehensweise bei der DSGVO-Kontrolle. Für die Zwecke, nach denen wir in der DSGVO-Kontrolle suchen, umfasst dies hauptsächlich die Überprüfung von Datenschutzrichtlinien, um sicherzustellen, dass sie grundlegende DSGVO-Anforderungen erfüllen. z. B. welche personenbezogenen Daten verarbeitet werden, was die Rechtmäßigkeit der Verarbeitung ist, indem sie auf rechte betroffener Personen hinweisen, wie ein Antrag einer betroffenen Person auf Zugriff (Subject Access Request, SAR) von einem Benutzer ausgeführt wird, wie der ISV SARs durchführt, die ISVs-Unternehmensdetails und Datenaufbewahrungsdetails.|
|Wir haben einen Penetrationstest durchlaufen; Wir haben jedoch keinen "sauberen" Penetrationstest, da wir keinen Penetrationswiederherstellungstest durchgeführt haben.  Müssen wir einen erneuten Test durchführen und einen sauberen Bericht haben? | Die Spezifikation Microsoft 365 Zertifizierung erfordert nicht, dass ISVs Penetrationstests durchführen müssen. Es können geeignete Korrekturnachweise bereitgestellt werden, um nachzuweisen, dass die im Penetrationstestbericht identifizierten Probleme behoben werden.|
|Einige der angeforderten Dokumentationen und Nachweise sind vertraulich. Gibt es Geheimhaltungsvereinbarungen (Non-Disclosure Agreements, NDAs) ? | Ja, einige der von Ihnen übermittelten Informationen sind öffentliche Informationen und einige möglicherweise vertrauliche Informationen.  Wenn Sie über eine vorhandene NDA mit Microsoft verfügen, gelten die Bedingungen dieses NDA für die vertraulichen Informationen, die Sie übermitteln.  Wenn Sie keine NDA mit Microsoft haben, gelten die Vertraulichkeitsbedingungen der Publisher Vereinbarung, die Sie im Partner Center signiert haben, für diese vertraulichen Informationen.|
|Wie übertragen wir vertrauliche Dokumentationen und Nachweise sicher im Rahmen der Microsoft 365 Zertifizierungsbewertung? | Derzeit verfügt Microsoft nicht über eine Plattform zum sicheren Teilen dieser Informationen. Es wird empfohlen, diese Informationen über die bereits eingerichteten sicheren Mechanismen freizugeben. Viele ISVs nutzen OneDrive und teilen einen authentifizierten Link zum Microsoft 365 Zertifizierungsteam.|
|Wir haben gerade einige zusätzliche Sicherheitsprozesse implementiert, um einige der Microsoft 365 Zertifizierungskontrollen zu erfüllen. Bedeutet dies, dass wir 12 Monate warten müssen, bevor wir zertifiziert werden können?|Nein, Microsoft erkennt an, dass Sie möglicherweise zusätzliche Sicherheitsprozesse entwickeln müssen, um Lücken zwischen vorhandenen Sicherheitsprozessen und dem, was von der Microsoft 365-Zertifizierung erwartet wird, zu schließen. Das Microsoft 365 Zertifizierungsteam überprüft neu entwickelte dokumentierte Prozesse und überprüft den Nachweis, dass der Prozess mindestens einmal durchgeführt wurde. Darüber hinaus sind keine historischen Nachweise erforderlich, da dies für diese neu entwickelten Prozesse nicht verfügbar ist. Nach zwölf Monaten wird dann mit der Bewertung einer Stichprobe von historischen Nachweisen während der jährlichen Bewertung begonnen.|
|Was bin ich für die Bereitstellung verantwortlich?| Während der Bewertung überprüfen Zertifizierungsanalysten die bereitgestellten Dokumente und Nachweise, um Ihre Konformität mit den Microsoft 365 Zertifizierungskontrollen zu bewerten. Im Rahmen dieser Arbeit fordert das Microsoft 365-Zertifizierungsteam Informationen an, die Architekturdetails, Diagramme, Datenspeicherdetails, App-Entwurfsdetails, Richtlinien- und Prozessdokumente, Konfigurationsdateien und Screenshots enthalten. In einigen Fällen oder wenn es für Sie einfacher ist, kann eine Screensharing-Sitzung eingerichtet werden, um die Zertifizierungsanalysten zu zeigen. Wenn vorhandene Compliance-Frameworks zur Unterstützung der Bewertungsaktivitäten verwendet werden sollen, ist eine angemessene Dokumentation erforderlich, die zeigt, was der externe Prüfer/Prüfer bewertet und bestätigt hat. Wenn die unterstützende Dokumentation nicht in der Lage ist, die erforderlichen Informationen bereitzustellen, um genau zu veranschaulichen, wie die Kontrollen innerhalb des externen Sicherheitsframeworks erfüllt wurden, kann das Microsoft 365 Zertifizierungsteam das externe Sicherheitsframework nicht zur Unterstützung der Microsoft 365 Zertifizierungsbewertung nutzen.|
|Muss ich zum Erreichen der Zertifizierung Änderungen an meiner aktuellen Infrastruktur vornehmen?| Es ist unwahrscheinlich, dass erhebliche Änderungen an der Infrastruktur erforderlich sind, um die Microsoft 365 Zertifizierung zu erfüllen. Die Kontrollen basieren auf bewährten Methoden für die Branchensicherheit und werden höchstwahrscheinlich bereits implementiert sein.  Wir haben in den meisten Fällen gesehen; ISVs mussten interne Prozesse aktualisieren, um Lücken zwischen den aktuellen Arbeitspraktiken und den Anforderungen innerhalb der Microsoft 365 Zertifizierung zu schließen. Wenn dies ein Problem ist, empfiehlt Microsoft, die neuesten Microsoft 365 Zertifizierungssteuerelemente zu überprüfen, die im Handbuch für Microsoft 365 Zertifizierungsübermittlungen zu finden sind, um sicherzustellen, dass Ihre derzeit bereitgestellte Umgebung und Ihre Arbeitspraktiken den definierten Steuerelementen entsprechen.|
|Hat Microsoft Empfehlungen zu bestimmten Komponenten/Infrastruktur/Software, die verwendet werden sollten, um die Zertifizierungsanforderungen zu erfüllen?| Microsoft gibt keine spezifische Empfehlung zu Lösungen, um Microsoft 365 Zertifizierungssteuerelemente zu erfüllen. Alle kommerziellen oder Open Source-Angebote können verwendet werden, sofern sie aktiv unterstützt und verwaltet werden.|
|Wie lange dauert es, bis die Bewertung abgeschlossen ist?| In der Regel kann es durchschnittlich 30 Tage dauern, bis eine Bewertung abgeschlossen ist, dies kann jedoch von vielen Variablen abhängen. Die Dauer der Fertigstellung kann je nach Größe der Hostingumgebung variieren, die zur Unterstützung der App/Add-In verwendet wird, der Art der Hostingumgebung, die die App/das Add-In unterstützt, und der Aufforderung, wie ISVs auf Nachweisanforderungen reagieren.|
|Wie viel Zeit muss ich diesem Prozess zuweisen? | Der Großteil der Arbeit besteht darin, die Dokumentation und die Nachweise zeitnah zu sammeln. Danach sollte es nicht mehr als ein paar Stunden pro Woche dauern, bis der Bewertungsprozess abgeschlossen ist. Einige Variablen, die sich auf die erforderliche Zeit auswirken können, sind: die Größe der Umgebung wirkt sich auf die Zeit aus, die für die Erfassung angeforderter Nachweise erforderlich ist, und ob externe Sicherheitsframeworks vorhanden sind, die zur Unterstützung der Bewertung genutzt werden können.  Wenn externe Sicherheitsframeworks vorhanden sind und angemessene unterstützende Dokumentation bereitgestellt werden kann, können Zertifizierungsanalysten diese externen Bewertungen verwenden, um eine Teilmenge der Microsoft 365 Kontrollen zu erfüllen, ohne dass Sie zusätzliche Nachweise bereitstellen müssen.|
|Warum gibt es einen festen Zeitrahmen von 60 Tagen für die Bewertung? | Wir haben einen Grenzwert für die Dauer festgelegt, für die eine Bewertung durchgeführt werden kann, da bereits erfasste Nachweise veraltet sein können, je länger eine Bewertung dauert.  Dies ist ein Zeitpunkt der Bewertung, daher muss ein geeigneter Zeitraum für den Abschluss zugewiesen werden. Nachdem Sie Ihre ursprüngliche Dokumentübermittlung übermittelt haben, werden wir mit einer Anfrage nach Nachweisen antworten. Der Zeitraum von 60 Tagen beginnt, wenn Sie den Nachweisantrag erhalten. Der [Leitfaden für Microsoft 365 Zertifizierungsübermittlungen](https://docs.microsoft.com/microsoft-365-app-certification/docs/certification-submission-guide) sollte gelesen werden, und Sie sollten sicher sein, dass alle Steuerelemente erfüllt werden können, bevor Sie Ihre ursprüngliche Nachweisübermittlung übermitteln.|
|Was geschieht, wenn die Bewertung nicht innerhalb des Zeitrahmens von 60 Tagen abgeschlossen ist? | Wenn die Bewertung während des Zeitrahmens von 60 Tagen nicht abgeschlossen ist, markiert Microsoft leider einen Fehler mit der Bewertung. Diese Markierung gilt nur für interne Statistiken und wird nie veröffentlicht. Sie können den Bewertungsprozess sofort neu starten. Sie werden jedoch aufgefordert, neue Nachweise erneut zu senden, um die neue Anwendung zu unterstützen.|
|Wie viel kostet mich die Microsoft 365 Zertifizierung? | Derzeit ist es KOSTENLOS für Sie, die Microsoft 365 Zertifizierung abzuschließen. |
|Was sind die Kosten für Penetrationstests im Rahmen dieses Programms? | Wenn Ihre App Penetrationstests unterzogen werden muss und dies nicht Teil Ihrer Sicherheitsaktivitäten ist, können Penetrationstests unter der Microsoft 365-Zertifizierung durchgeführt werden und sind kostenlos. Der Umfang der Penetrationstests ist auf die App und die unterstützende Infrastruktur beschränkt, die sich im Bereich der Microsoft 365-Zertifizierung befindet. |
|Verfügen Sie über Marketingmaterialien, die verwendet werden können, um dafür zu werben, dass unsere App zertifiziert wurde? | Wir bieten derzeit keine Werbe- oder Marketingmaterialien an. Wir empfehlen Ihnen jedoch, Kunden mitzuteilen, dass Ihre App eine Microsoft 365 Zertifizierung erhalten hat. Die einzigen verfügbaren Grafiken sind jedoch die Grafiken, die in Ihrem App-Eintrag in AppSource und auf den Microsoft Docs-Seiten angezeigt werden. Dieses Dokument wird aktualisiert, wenn Marketingmaterialien verfügbar werden.
|Nach welchem Nachweisgrad suchen Sie bei der Durchführung der Bewertung? | Nachweise, die während der Microsoft 365 Zertifizierungsbewertung bereitgestellt werden, müssen genügend Sicherheit bieten, dass Sie die spezifischen Microsoft 365 Zertifizierungssteuerelemente erfüllen, die bewertet werden.  Nachweise können in Form von Konfigurationsdateien, Screenshots von Einstellungen oder Nachweisen, Richtlinien-/Prozedurdokumentation oder Screensharing-Sitzungen vorliegen, um dem Zertifizierungsanalysten Nachweise zu demonstrieren.  Im Folgenden finden Sie zwei Beispiele:|
||**Bewertungsaktivität:**"Demonstrieren, dass Antivirensoftware in allen beispielweisen Systemkomponenten ausgeführt wird." – Für dieses Steuerelement können Sie einen Screenshot von jedem Gerät im Beispiel bereitstellen, das Den Virenschutz unterstützt, der den laufenden Antivirenprozess zeigt, oder wenn Sie über eine zentrale Verwaltungskonsole für Antivirensoftware verfügen, können Sie ihn möglicherweise über diese Verwaltungskonsole veranschaulichen.|
||**Bewertungsaktivität:**"Demonstrieren, wie neue Sicherheitsrisiken identifiziert werden." – Dieses Steuerelement stammt aus dem Abschnitt "Patchverwaltung". Sie möchten, dass Sie über einen formell dokumentierten Prozess verfügen, mit dem Sie neue Sicherheitsrisiken identifizieren.  Dies kann sich in Ihrem Quellcode befinden, muss sich aber auch innerhalb der Supportumgebung befinden, z. B. Windows Sicherheitsrisiken, Sicherheitsrisiken innerhalb von Webabhängigkeiten (z. B. AngularJS, JQuery usw.).  Sie sollten über einen dokumentierten Prozess verfügen, dem Sie folgen, um neue Sicherheitsrisiken zu identifizieren. Stellen Sie daher das dokumentierte Prozessdokument bereit.  Zusätzlich zur Dokumentation müssten Sie nachweisen, dass der Prozess befolgt wird. Wenn Sie z. B. eine npm-Überwachung verwenden, um Abhängigkeiten auf Sicherheitsrisiken zu überprüfen, liefert die Bereitstellung eines Beispiels von Berichten Nachweise.  Wenn Sie mehrere Prozesse verwenden, z. B. für unterschiedliche Systemkomponenten, müssen Alle Prozesse nachgewiesen werden.|
